# 台本ブラッシュアップスキル

`research.md` と `route.md` をもとに `script.md` の初稿を作り、フィードバックを受けて反復的に磨き込む。

## 前提

- 完成例として `stations/2026-07-18_川口・あんぎょう/script.md` を参照する（構成の分け方、ナレーションの文体、出典・撮影メモの残し方の見本にする）
- 台本の骨格は `_template/script.md` の見出し（オープニング／徒歩パート／道の駅到着〜館内紹介／グルメ・特産品紹介／エンディング）だが、上記の完成例のようにルートの区間ごとにパートを細分化してよい

## 手順

### 1. 対象の特定

- `$ARGUMENTS` またはユーザーの発言から対象の `stations/YYYY-MM-DD_道の駅名/` フォルダを特定する
- 対象が曖昧な場合は `stations/` 配下のフォルダ一覧を提示し、AskUserQuestionで確認する
- `research.md` の「見どころ・地域の取り組み」に採用済みトリビアが無い（`research-station` のステップ5が未実施）場合は、先にそちらを済ませるかどうかユーザーに確認する

### 2. インプットの読み込み

- `research.md`（基本情報・アクセス・グルメ・採用済みトリビアと出典）
- `route.md`（区間ごとの徒歩ルート・撮影ポイント・注意点）
- `docs/learnings.md` の「台本：フィードバックの傾向」セクション（存在すれば）を読み込み、過去に繰り返し指摘された傾向を初稿の時点から反映する

### 3. 初稿作成

- `route.md` の区間・`research.md` の採用済みトリビア・グルメ情報を組み合わせて `script.md` を構成する
- ナレーションは実際に読み上げる文章として書く（体言止めの箇条書きで終わらせない）
- データ・トリビアの直後に出典を明記する。制作用の注意点（未確認事項・裏取りメモ・撮影上の注意）は完成例の「📝 撮影メモ」「📝 構成メモ」のように本文のナレーションと別枠に分けて残す
- 想定尺と、パートごとの尺配分の目安表を含める

### 4. フィードバックループ

- 初稿を提示し、ユーザーからのフィードバックを受ける
- フィードバックに沿って該当パートのみ修正する（指摘されていない箇所まで書き直さない）
- 修正のたびに、フィードバックの要旨と対応内容を控えておく（ステップ5でまとめて記録するため）
- ユーザーが完了を示す（「OK」「これでいい」等）まで反復する

### 5. 学習の記録

- `docs/learnings.md` の「台本：フィードバックの傾向」個別ログに、今回の道の駅名を見出しにして、フィードバック内容と対応をまとめて追記する
- 個別ログ全体を見返し、同じ種類の指摘（例：「オープニングの尺を短く」「年表を並べず一番意外な事実に絞る」等）が2回以上確認できた場合のみ「繰り返し指摘されるポイント」を更新する。1回限りの指摘は個別ログに残すだけに留める

### 6. 完了報告

- 完成した `script.md` の構成・想定尺を報告する
- `docs/learnings.md` に追記した内容（個別ログ・傾向の更新有無）を報告する

## スコープ外

- `research.md`・`route.md` の作成自体（`research-station` や通常の依頼で行う）
- 動画編集・撮影そのもの
- コミット・プッシュ・PR作成は、ユーザーの指示があるまで行わない
