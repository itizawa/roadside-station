# 道の駅スケジューリングスキル

道の駅動画の制作フロー（訪問→編集→投稿）に沿って、Googleカレンダーに3種類の予定を作成する。

## 前提

- コンセプト上、投稿頻度は週1回（[README.md](../../../README.md) 参照）
- 作業時間帯のデフォルトは **22:00〜24:00**（ユーザーの発言や過去のやり取りで別の時間帯が示されていればそちらを優先する）
- 週一投稿の定番曜日・時間がまだ決まっていない場合は、以下を目安に提案する
  - **金曜 19:00〜21:00**（推奨）：週末のお出かけ先を探すタイミングで視聴・保存されやすい
  - 土曜 8:00〜10:00：これから出かける層に刺さりやすいが、編集を金曜深夜までに終える必要がありシビア
  - 一度決まった曜日・時間はプロジェクトの定番として以後もそのまま使う（ユーザーから変更の指示がない限り、毎回提案し直さない）

## 手順

### 1. 対象の道の駅と訪問日の特定

- `$ARGUMENTS` またはユーザーの発言から、対象の道の駅（`stations/YYYY-MM-DD_道の駅名/` フォルダ）を特定する
- フォルダ名から訪問日と道の駅名を読み取る。複数の道の駅が同日訪問の場合（例：`2026-07-26_いちかわ` と `2026-07-26_やちよ`）はまとめて1つの訪問予定にしてよい
- 対象が曖昧な場合は `stations/` 配下のフォルダ一覧を提示し、AskUserQuestion でどれを対象にするか確認する

### 2. 既存の投稿曜日・時間帯の確認

- Google Calendar MCP（`mcp__claude_ai_Google_Calendar__list_events`）で直近の「◯◯ 動画公開」イベントを検索し、定番の曜日・時間が既にあるか確認する
- 見つからない場合は前提セクションの提案をもとに AskUserQuestion で確認する

### 3. 必要な情報の確認（AskUserQuestion）

以下が会話やカレンダーから確定できない場合は、推測で進めず質問する。

- **編集作業をどの日に行うか**（複数日可。例：平日毎日、特定の曜日3日など）
- **投稿日**（週一投稿の定番枠に当てはめる。8月第一週など期間指定の場合は候補日を2〜3個提示する）

### 4. Google Calendar MCPツールのロード

未ロードの場合、以下をまとめてロードする。

```
ToolSearch query: "select:mcp__claude_ai_Google_Calendar__list_events,mcp__claude_ai_Google_Calendar__create_event,mcp__claude_ai_Google_Calendar__list_calendars"
```

### 5. イベント作成

`mcp__claude_ai_Google_Calendar__create_event` で以下を作成する（`timeZone` は `Asia/Tokyo` を指定）。

| 予定 | タイトル | 日時 |
|------|---------|------|
| 訪問予定 | `道の駅◯◯ 訪問` | 訪問日、終日（`allDay: true`） |
| 編集予定 | `◯◯ 動画編集` | 指定された各日、22:00〜24:00（終了時刻は日をまたぐため翌日 `00:00:00` を指定する） |
| 投稿予定 | `◯◯ 動画公開` | 確定した投稿日時、30分間 |

- 既に同名・同日のイベントが存在する場合は重複作成せず、ユーザーに確認する
- `endTime` に `24:00:00` は指定できない（ISO 8601エラーになる）。日をまたぐ場合は翌日の日付で `00:00:00` を使う

### 6. 完了報告

- 作成したイベントを一覧で報告する（訪問予定・編集予定・投稿予定）
- 投稿の定番曜日・時間を新たに決めた場合は、その旨と理由を添える

## スコープ外

- カレンダーイベントの削除・変更は、ユーザーから明示的な指示があるまで行わない
- 動画編集・投稿作業そのもの（このスキルはスケジュール登録のみを行う）
